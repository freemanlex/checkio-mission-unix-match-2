Іноді ти опиняєшся в ситуації, коли серед величезної кількості файлів на всьому комп'ютері або в окремій теці потрібно знайти файли певного типу. Наприклад, зображення з розширенням '.jpg' або документи з розширенням '.txt', або навіть файли, що містять слово 'метелик' у імені. Робити це вручну може зайняти багато часу. "Пошук відповідності" або шаблони для пошуку файлів за певною маскою - це саме те, що тобі потрібно для подібних проблем.
Ця місія допоможе тобі зрозуміти, як це працює.

Тобі потрібно з'ясувати, чи відповідає даний Unix шаблон даному імені файлу.

Ось невелика таблиця, в якій подані символи, які можна використовувати в шаблонах.

[seq] відповідає будь-якому одному символу в seq, наприклад [123] значить - '1' або '2' або '3'
[!seq] відповідає будь-якому одному символу крім тих, що в seq, наприклад [!123] значить крім - '1' і '2' і '3'
[] сукупність без символів ніколи не матиме відповідності

Зауваж, що вираз в одній парі квадратних дужок відповідає лише 1 символу. Тож: {% if is_js %} 

unixMatch('0123', '[!abc]123') == true;
// але
unixMatch('00123', '[!abc]123') == false;
{% else %} 
unix_match('0123', '[!abc]123') == True
# але
unix_match('00123', '[!abc]123') == False
{% endif %}






    Вхідні дані: Два аргументи. Обидва рядки.





    Вихідні дані: Логічне (булеве) значення.
